What it is

Unity AI Gateway

1
AI Assets (UC) Models, functions, connections, MCP Services as UC securables
2
AI Traffic Control Rate limits, spend caps, usage tracking, inference logging, fallbacks
3
AI Service Behavior Service policies (ABAC) governing content, tool calls, and approvals
Three things to understand
It is a traffic layer, not a model layer.
The model doesn't change. What changes is what happens before and after every call.
It is UC-governed.
Endpoints, MCP Services, prompts, and agent services are Unity Catalog securables. Same permissions model you use for tables and functions.
Guardrails are a subset of service policies.
Service policies can also govern tool calls and require human approval — guardrails alone cannot.
Architecture

Three products. One gateway.

Capability 1 of 8

Serving Endpoint Routing

Route different products to different foundation models through a single gateway surface. Change models without touching application code.

Customer Genie

Routes to databricks-meta-llama-3-3-70b-instruct

Cost-optimized. Retail Q&A workload. High query volume, low complexity.

Analyst Assistant

Routes to databricks-claude-sonnet-4

Premium reasoning. Equity research. Lower volume, higher value per call.

Compliance Agent

Routes to databricks-gpt-4o

Batch overnight. Structured output required. Fallback enabled on 429/5XX.

What the gateway controls

  • Traffic split across multiple backend entities per endpoint
  • Fallback to secondary provider on 429 or 5XX errors
  • Provider switching without application code changes
  • Native API support: OpenAI, Anthropic Messages, Gemini

Acme impact

Finance moved Analyst Assistant from GPT-4o to Claude Sonnet 4 by changing one gateway config. Zero code changes in the application. Zero downtime. Fallback covers model outages automatically.

Acme Financial

Customer Genie

  • Custom block — blocks any mention of competitor products in input
  • PII Redaction — account numbers replaced with [ACCOUNT] in model output before display
  • Evaluator: databricks-gpt-5-nano

Compliance Agent

  • Custom block (output) — enforces structured JSON response shape; flags free-form text
  • Hallucination block — output guardrail catches invented regulation citations

Fail-closed: evaluator timeout or parse error blocks the request. Use mode=LOG for dry-run testing on live traffic.

Capability 4 of 8

Service Policies (ABAC)

A governance layer distinct from guardrails. Guardrails govern content. Service policies govern access — who can call what, under what conditions, with what approval.

AspectGuardrailsService Policies
ScopeContent filteringContent + tool calls + identity
ActionsBlock or sanitizeAllow, deny, require approval
Tool calls—✓ Can block out-of-policy calls
Human-in-loop—✓ Supported
Attaches toGateway endpointsMCP Services & Model Services

Relationship: Guardrails are built-in service policies. Custom service policies extend beyond them — governing tool invocation and approval workflows that guardrails cannot reach.

Acme Financial

Analyst Assistant

  • Only Senior Analyst group can invoke the equity valuation tool
  • Junior analysts: read-only tool access via service policy deny on write operations
  • Policy attached to the MCP Service securable, not the application

Compliance Agent

  • Any report that changes a flag status requires supervisor approval before proceeding
  • Human-in-the-loop gate: agent pauses, approval record written to audit log, then continues
  • Approval enforced at the gateway layer — cannot be bypassed by application code

Capability 6 of 8

Usage Monitoring

Every AI call lands in system.ai_gateway.usage. Account-level, admin-readable. 30+ columns. The authoritative source for cost attribution, anomaly detection, and compliance reporting.

Key columns
ColumnWhat it holds
endpoint_nameWhich product called
requesterUser email / SP ID
destination_modelWhich model served it
input_tokens / output_tokensToken counts
latency_msEnd-to-end latency
status_code200 passed, 400 blocked, 429 rate-limited
routing_informationFallback attempts with latency
Acme queries
-- Daily spend by product
SELECT endpoint_name,
  DATE(event_time) AS day,
  SUM(input_tokens + output_tokens) AS tokens
FROM system.ai_gateway.usage
GROUP BY 1, 2
ORDER BY day DESC;
-- Anomaly: >2x rolling avg
SELECT requester, COUNT(*) AS calls
FROM system.ai_gateway.usage
WHERE event_time > NOW() - INTERVAL 1 HOUR
HAVING calls > 2 * (
  SELECT AVG(hourly_calls) FROM avg_by_user
);

Capability 8 of 8

Prompt Registry

Versioned prompt templates stored as Unity Catalog functions (via MLflow Prompt Registry). Immutable per version. Audit trail on every change. Governed by UC permissions per schema.

How it works

  • Each prompt stored as a UC function: catalog.schema.prompt_name
  • Versions are immutable — edit creates a new version, old ones remain
  • Variables use {{variable}} syntax; resolved at call time
  • UC lineage tracks who changed what and when
# Load pinned version at runtime
import mlflow

prompt = mlflow.load_prompt(
  "acme.compliance.regulatory_report",
  version=7  # pinned — never drifts
)
response = client.chat.completions.create(
  model="compliance-agent",
  messages=[{
    "role": "system",
    "content": prompt.format(
      jurisdiction="SEC",
      report_date=today
    )
  }, ...]
)
Acme Financial — Compliance Agent

regulatory_report prompt

  • Stored at acme.compliance.regulatory_report
  • Production pin: v7
  • v8 under review by compliance team before promotion
  • Rollback: change version pin, redeploy — 30 seconds

Audit trail

  • UC lineage records every version change: who, when, from where
  • Regulator request: "show me the exact prompt used on March 14" — answered from UC lineage in seconds
  • Schema permissions: only compliance-admins group can register new versions
